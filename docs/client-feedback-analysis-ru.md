| Критерий | ЮKassa | Т-Банк | CloudPayments |

| Методы | Все (карты, СБП, Apple/Google Pay) | Все | Все |
| Онлайн-касса 54-ФЗ | ✅ Встроенная | ⚠️ Отдельно | ⚠️ Отдельно |

| Для Pre-MVP | ✅ Идеально | ✅ Хорошо | ⚠️ Избыточно |

**Вывод:** ЮKassa - оптимальный баланс простоты, функциональности и стоимости.

### 5.2 Pre-MVP (приоритетный запуск)

Клиент хочет **Pre-MVP в ближайшее время** с упрощенными тарифами:

| Тариф | Цена | Себестоимость | Маржа |
|-------|------|---------------|-------|
| Базовый | 100₽ | ~1.5-5₽ | ~95-98% |
| Пакет 5 | 300₽ | ~7.5-25₽ | ~92-97% |
| PRO | 500₽ | ~3-10₽ | ~98% |
| Кассандра | 1000₽ | ~5-15₽ | ~98% |

**Наша оценка:** Цены экономически выгодны. Рекомендуем реализовать Pre-MVP в течение 2-3 недель.

### 5.3 Архитектурные рекомендации

1. **Единый backend API** - критично для синхронизации лендинга и Telegram
2. **Supabase Auth как SSO** - уже используем, расширяем на Telegram
3. **PostgreSQL для единой истории** - уже есть база, добавляем таблицы
4. **Webhook-based биллинг** - ЮKassa отправляет события, мы обновляем квоты

### 5.4 Что мы хотим изменить/добавить (как разработчики)

| Предложение | Обоснование |
|-------------|-------------|
| **Error recovery UI** | Клиент отметил риск "быстрая реакция на сбои". Добавим retry-механику с красивым UX |
| **Rate limiting с grace period** | Для FREE: если 119 часов прошло, показываем таймер, а не блокируем сразу |
| **Offline-first для истории** | PWA с кэшем - пользователь видит историю даже без сети |
| **A/B тестирование цен** | Для Pre-MVP: возможность быстро менять цены без деплоя |
