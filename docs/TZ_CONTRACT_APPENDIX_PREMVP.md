## 1. ОПИСАНИЕ ЭТАПА PRE-MVP

Разработка минимальной версии платформы для **тестирования гипотезы** и проверки бизнес-модели. Pre-MVP создается для сбора обратной связи от ранних пользователей и валидации концепции AI-анализа кофейной гущи.

**Цели этапа:**
- Проверить востребованность сервиса на реальной аудитории
- Отладить промпты для AI-анализа изображений
- Собрать примеры реальных запросов пользователей
- Получить первичные метрики (конверсия, время использования, отзывы)

**Технологический подход:**
- Backend API для обработки запросов
- Telegram Bot как точка входа для пользователей
- Интеграция с AI-сервисами для анализа изображений
- Упрощенная архитектура без постоянного хранилища данных

## 2. ФУНКЦИОНАЛ PRE-MVP

### 2.1. Основные возможности

**Telegram Bot интерфейс:**
- ✅ Прием команд от пользователей через Telegram
- ✅ Прием фотографий кофейных чашек
- ✅ Отправка результатов анализа пользователю
- ✅ Базовая навигация (старт, помощь, информация)

**Backend обработки:**
- ✅ API для приема изображений от Telegram бота
- ✅ Валидация и предобработка изображений
- ✅ Интеграция с AI-сервисами (Vision API для распознавания, Text API для интерпретации)
- ✅ Генерация структурированного психологического анализа
- ✅ Форматирование ответа для удобного чтения в Telegram

**AI-анализ:**
- ✅ Распознавание паттернов на изображении кофейной чашки
- ✅ Генерация психологической интерпретации (5 тематических блоков)
- ✅ Поддержка различных фокус-областей (общий, карьера, отношения, благополучие)
- ✅ Адаптация тона и стиля ответа под целевую аудиторию

### 2.2. Ограничения Pre-MVP

**Сознательно НЕ реализовано** (будет в основном MVP):
- ❌ База данных и хранение истории пользователей
- ❌ Система подписок и тарифных планов
- ❌ Платежные интеграции
- ❌ Персонализация на основе предыдущих анализов
- ❌ Admin панель и аналитика
- ❌ Масштабируемая инфраструктура

## 5. ПОРЯДОК СДАЧИ-ПРИЕМКИ РАБОТ

### 5.1. Процедура приемки Pre-MVP

1. **Демонстрация функционала** (14.10.2025):
   - Исполнитель проводит демонстрацию работающего Pre-MVP заказчику
   - Передает тестовый доступ к Telegram боту
   - Предоставляет инструкцию по использованию

2. **Тестирование заказчиком** (14.10.2025):
   - Заказчик проводит функциональное тестирование (минимум 5 тестовых запросов)
   - Проверяет соответствие функционала разделу 2.1

3. **Приемка или замечания** (14.10.2025):
   - При успешном тестировании: подписание Акта приемки
   - При наличии критических замечаний: предоставление списка (в течение 2 часов после демо)

4. **Устранение замечаний** (при необходимости):
   - Исполнитель устраняет критические замечания в течение 24 часов
   - Повторная приемка по аналогии с п. 2-3

### 5.2. Критерии успешной приемки

✅ **Обязательные условия:**
- Telegram бот отвечает на команды и принимает фотографии
- AI успешно анализирует загруженные изображения кофейных чашек
- Результат анализа корректно форматируется и отправляется пользователю
- Время обработки одного запроса не превышает 60 секунд
- Отсутствуют критические ошибки (падения бота, потеря запросов)

✅ **Что передается заказчику:**
- Рабочий Telegram бот (активный и доступный для тестирования)
- Доступ к backend серверу (при необходимости)
- Краткая инструкция по использованию и тестированию
- Описание настроенных AI промптов

## 6. ГАРАНТИИ И ОТВЕТСТВЕННОСТЬ

### 6.1. Гарантийные обязательства

**Срок поддержки Pre-MVP:** до момента запуска основного MVP (ориентировочно до 01.02.2026)

**В течение этого периода исполнитель обязуется:**
- Поддерживать работоспособность Pre-MVP (доступность бота, обработка запросов)
- Исправлять критические ошибки в течение 24 часов с момента сообщения
- Консультировать по использованию системы

**Не входит в гарантийные обязательства:**
- Добавление нового функционала
- Изменение логики AI-анализа (кроме исправления явных ошибок)
- Увеличение лимитов обработки запросов

### 6.2. Ограничение ответственности

Pre-MVP является **экспериментальной версией** для тестирования гипотезы. Исполнитель не несет ответственности за:
- Качество AI-генерируемого контента (субъективность интерпретаций)
- Недовольство конечных пользователей результатами анализа
- Технические сбои сторонних сервисов (Telegram API, AI API)
- Превышение лимитов бесплатных/платных API сторонних сервисов
